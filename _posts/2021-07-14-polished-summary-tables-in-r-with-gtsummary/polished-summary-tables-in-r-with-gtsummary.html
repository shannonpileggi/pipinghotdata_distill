TL; DR

The gtsummary package in R creates amazing publication / presentation / whatever-you-need-it-for ready tables of summary statistics. Try it out!

The best, quickest table

To get a quick summary table, first retain only the variables you want to see in the summary table. In this case, I removed record to avoid summary statistics of an ID. Next, because the imported data has value labels from the .sav data file, I converted those variables to a factor for summarizing.

dat %>% 
  # remove from summary table
  dplyr::select(-record) %>% 
  # convert labelled values to a factor ----
  mutate_at(vars(q12), haven::as_factor) %>% 
  # create a table with columns for year ----
  gtsummary::tbl_summary(by = year)

And wait - did you see that?! The raw data had variable names of q12, stheight, but the table printed the variable label! (I previously tweeted about the awesome package pairing of haven and gtsummary.) If your data does not come with handy labels, you can create them with the label option in tbl_summary or with the var_label function in the labelled package.

A few modifications

Here are a few modifications you might be interested in trying to customize your table.

dat %>% 
  # remove from summary table
  dplyr::select(-record) %>% 
  # covert labelled values to a factor ----
  mutate_at(vars(matches("q")), haven::as_factor) %>% 
  tbl_summary(by = year,
    # custom statistic formats ----
    statistic = list(all_continuous() ~ "{mean} ({sd})",
                     all_categorical() ~ "{n} / {N} ({p}%)"), 
    # don't list missing data separately
    missing = "no" 
  ) %>%
  add_n() %>% # add column with total number of non-missing observations
  add_p() %>% # test for a difference between groups
  add_overall() %>% # add an overall column
  bold_labels()

Discrete numeric values

One default I frequently correct is treatment of discrete numeric values. For example, consider a rating scale with possible values of 1, 2, 3, … 7, but in which respondents only select values of 3, 4, 5. Making an educated guess and only seeing three unique values, gtsummary will treat this as a categorical variable and return frequencies of those values; however, you may still want a mean. You can override these potentially undesirable defaults in gtsummary 🙌.

This data set does not have a great example of this, so I’ll make one. I am going to pretend that q12 is numeric for demonstration,

dat %>% 
  # keep q12 for summary table ----
  dplyr::select(q12) %>% 
  # convert labelled values to numeric by removing value labels ----
  mutate_at(vars(q12), haven::zap_labels) %>% 
  # subtract one to make pretend example more realistic ----
  mutate_at(vars(q12), ~ . -1) %>% 
  # create a table with columns for year ----
  gtsummary::tbl_summary()
Characteristic N = 30,3891
Weapon carrying
0 21,924 (84%)
1 843 (3.2%)
2 978 (3.7%)
3 368 (1.4%)
4 2,048 (7.8%)
Unknown 4,228

1 Statistics presented: n (%)

In this table, a 4 represents carrying a weapon to school 4 days in the past month. Notice the default is provide a frequency summary as if it is a categorical variable due the few values present. If instead you want a mean to summarize this variable, specify the variable as continuous with the type argument in tbl_summary.

dat %>% 
  # keep q12 for summary table ----
  dplyr::select(q12) %>% 
  # convert labelled values to numeric by removing value labels ----
  mutate_at(vars(q12), haven::zap_labels) %>% 
  # subtract one to make pretend example more realistic ----
  mutate_at(vars(q12), ~ . -1) %>% 
  # create a table with columns for year ----
  gtsummary::tbl_summary(
    # treat q12 variable as continuous ----
    type = list(q12 ~ "continuous"),
    # custom statistic format ----
    statistic = list(all_continuous() ~ "{mean} ({sd})")
  )
Characteristic N = 30,3891
Weapon carrying 0.46 (1.16)
Unknown 4,228

1 Statistics presented: Mean (SD)

Missing data

Before we part, let’s make sure we understand how the package handles missing data and our options.

dat_raw %>% 
  filter(year == 2017) %>% 
  dplyr::select(q69) %>% 
  mutate_at(vars(q69), haven::as_factor) %>% 
  gtsummary::tbl_summary()
Characteristic N = 14,7651
Weight loss
Lose weight 5,462 (47%)
Gain weight 2,247 (19%)
Stay the same weight 1,833 (16%)
Not trying to do anything 1,999 (17%)
Unknown 3,224

1 Statistics presented: n (%)

From this summary table, I can infer that there are 14,765 records in the year 2017; 11,541 of them have non-missing values and 3,224 have missing values. The percents shown in the table are the percent of the non-missing base n; that is, 5,462 / 11,541 is 47%.

You can remove printing of the count of unknown values if you like, which does not change any of other numbers in your table.

dat_raw %>% 
  filter(year == 2017) %>% 
  dplyr::select(q69) %>% 
  mutate_at(vars(q69), haven::as_factor) %>% 
  gtsummary::tbl_summary(
    # supress printing count of unknown values ----
    missing = "no"
  )
Characteristic N = 14,7651
Weight loss
Lose weight 5,462 (47%)
Gain weight 2,247 (19%)
Stay the same weight 1,833 (16%)
Not trying to do anything 1,999 (17%)

1 Statistics presented: n (%)

If you want your n at the top of the column to reflect the n for non-missing observations, I would remove those in your data cleaning process.

dat_raw %>% 
  filter(year == 2017) %>% 
  dplyr::select(q69) %>%
  # remove all observations with missing values ----
  drop_na() %>% 
  mutate_at(vars(q69), haven::as_factor) %>% 
  gtsummary::tbl_summary()
Characteristic N = 11,5411
Weight loss
Lose weight 5,462 (47%)
Gain weight 2,247 (19%)
Stay the same weight 1,833 (16%)
Not trying to do anything 1,999 (17%)

1 Statistics presented: n (%)

On the other hand, if the missing data represented a valid category that you wanted counted, you could replace the missing values (which would shift the percentages).

dat_raw %>% 
  filter(year == 2017) %>% 
  dplyr::select(q69) %>%
  mutate_at(vars(q69), haven::as_factor) %>% 
  # replace missing values for factor ----
  mutate_at(vars(q69), fct_explicit_na, "Missing weight action") %>% 
  gtsummary::tbl_summary()
Characteristic N = 14,7651
Weight loss
Lose weight 5,462 (37%)
Gain weight 2,247 (15%)
Stay the same weight 1,833 (12%)
Not trying to do anything 1,999 (14%)
Missing weight action 3,224 (22%)

1 Statistics presented: n (%)

Limitations

There really aren’t many! The gtsummary package has readable output that is easily customizable. The one hang up for me is that I work with many who prefer to see tables in excel, and this an output not currently supported in gtsummary (see supported output types). No shade here, I’m waiting patiently for this to happen in the gt package. 😁
